Patron recomendado

Si una pantalla como /venta abre un modal para editar proveedores, el componente del modal puede encargarse de validar y guardar. El footer solo llama metodos como save() mediante componentInstance.

El componente padre queda limpio: abre el modal y, cuando este se cierra, decide si refresca datos o actualiza su vista.
